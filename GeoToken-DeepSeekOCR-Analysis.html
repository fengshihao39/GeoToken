GeoToken × DeepSeek-OCR

可迁移与不可迁移的系统分析

遥感 3D · 几何保持视觉分词 · 跨视角压缩

1 核心判断

可以迁移思想，不能直接迁移方法。 更准确的研究表述应当是：
"Inspired by DeepSeek-OCR's optical compression, we study geometry-preserving visual tokenization for remote sensing 3D, where multi-view observations are compacted into compact tokens while preserving correspondence, disparity, and metric 3D geometry."

DeepSeek-OCR 的关键结论：把长文本映射到二维图像后，可以用少量 vision tokens 承载大量文本信息。论文中，text tokens / vision tokens 小于 10× 时 OCR 解码精度约 97%，到 20× 时还有约 60%。这说明视觉 token 是高密度信息载体，也说明压缩是有边界的。

迁移到 GeoToken 时，最重要的判断是：DeepSeek-OCR 压的是"二维可读信息"；GeoToken 要压的是"多视角几何证据"。 这两者不是同一个问题。

2 本质差异：OCR vs 遥感 3D

DeepSeek-OCR 压缩目标

  • 一页文字在图像里是固定的 2D 排布
  • 模型只需要从图中恢复离散文字
  • 局部模糊时，语言模型可凭上下文补全

GeoToken 必须保留的信息

  • 同名点 correspondence
  • 视差 disparity & 射线 ray
  • 相机 / RPC 几何
  • 高度 / 深度连续值
  • 边缘和结构线
  • 遮挡关系
  • 多视角一致性

这些几何信息一旦在压缩阶段丢失，后面的 3D decoder 很难凭语义补回来。所以 GeoToken 不能是普通 visual tokenization，而必须是 geometry-preserving visual tokenization。

3 可以借鉴的三类结构

① 高分辨率输入 + 中间 compressor + 少量 token 输出

DeepSeek-OCR 的 DeepEncoder 设计目标是在高分辨率输入下保持较低 activation，并把视觉信息压到可控数量的 vision tokens。对遥感也类似：原始影像很大，必须压缩，否则 token 成本爆炸。

② 多档 token budget 的可扩展设计

DeepSeek-OCR 用不同分辨率和 token 数测试压缩率与恢复精度的关系。GeoToken 也可以建立类似的曲线：

# 预期的 scaling law
token 数量 ↓  →  DSM / depth / point cloud 误差 ↑

从而证明：在多少 token 下，几何信息还能保持到什么程度。

③ encoder-decoder 验证范式

DeepSeek-OCR 用 OCR 恢复精度验证 vision token 是否保留了文本信息。GeoToken 可以定义对应的几何度量：

DSM 误差 depth 误差 reprojection error Chamfer distance photometric consistency

4 不能迁移的三个限制

① 不能单张图独立压缩

DeepSeek-OCR 处理的是单页图像；遥感 3D 依赖多视角匹配。每个视角各自压成 tokens 再让 decoder 自己去猜对应关系，风险很高。

错误路线
image1 → tokens1
image2 → tokens2
image3 → tokens3
         ↓
  decoder 自己学 3D
合理路线
multi-view images + camera/RPC
         ↓
geometry-aware cross-view aggregation
         ↓
   compress into geometry tokens
         ↓
       3D decoder

跨视角对应关系应该在压缩前或压缩过程中建立，而不是压缩后才补救。

② 不能只用普通 2D position embedding

OCR 的位置编码主要服务于页面文字位置；GeoToken 的位置编码必须服务于三维几何。token 里至少要包含：

pixel coordinate camera I/E / RPC ray direction GSD / scale height hypothesis geo-coordinate prior viewing angle

否则 token 可能知道"这里像建筑"，但不知道"这个建筑边缘对应哪条空间射线、可能在哪个高度"。

③ 不能只用重建 / 语义 loss

OCR 解码的是离散文本；GeoToken 解码的是连续几何。训练目标必须加入度量约束：

depth / height loss DSM loss multi-view reprojection loss photometric consistency epipolar matching loss normal / edge consistency Chamfer distance uncertainty loss

否则 token 可能保留语义，却不保留几何精度。

5 可行的 GeoToken 设计

# ─── GeoToken 整体架构 ───

Multi-view remote sensing images
         + RPC / camera geometry
         + GSD / geo metadata
               ↓
        Dense image encoder
               ↓
     Ray-aware positional encoding
               ↓
     Epipolar / plane-sweep / cost-volume aggregation
               ↓
     Geometry-preserving token compressor
               ↓
           GeoTokens
               ↓
DSM / depth / point cloud / Gaussian / mesh decoder
⚡ 最关键的是中间两步：ray-aware positional encoding 和 geometry-aware cross-view aggregation。没有这两步，所谓"借鉴 DeepSeek-OCR"就容易变成普通图像压缩，对 3D 重建帮助有限。

6 最终判断

可以迁移思想，不能直接迁移方法。

这个方向是成立的，而且有研究价值。

真正的难点不在"能不能压缩"，而在于——

压缩后的 token 是否仍然可匹配、可定位、可恢复连续三维坐标？

这，才是 GeoToken 真正的核心问题。
